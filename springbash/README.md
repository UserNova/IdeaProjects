# Objectifs : 

Ce TP a pour objectif de permettre à l’étudiant de maîtriser l’automatisation du cycle de vie d’une application Spring Boot à l’aide de scripts Bash.

Il vise le développement des compétences suivantes :

- Compréhension du cycle complet de développement et d’exécution d’une application Spring Boot (construction, exécution, arrêt, déploiement).
- Utilisation de scripts Bash pour automatiser les opérations de build, de lancement, de supervision et de déploiement.
- Application de concepts DevOps tels que la livraison continue et le déploiement reproductible.
- Manipulation de notions systèmes : processus, PID, permissions, flux d’entrée et de sortie.
  
Ce travail relie les compétences de développement logiciel et celles d’administration Linux, nécessaires dans les environnements professionnels de déploiement continu.

# Contexte : 

Dans un environnement de production, les applications Spring Boot ne sont pas exécutées depuis un IDE mais via des scripts ou des outils d’intégration continue.
Le développeur doit être capable de :

- Générer un fichier exécutable (JAR) de l’application.
- Le transférer sur un serveur Linux.
- Automatiser le lancement, l’arrêt et la supervision du service.
  
Ce TP montre comment réaliser ces opérations à travers des scripts Bash structurés, robustes et réutilisables.

# Outils et technologies mobilisés : 

- Spring Boot : framework Java pour les applications autonomes.
- Maven : outil de build et de gestion de dépendances.
- Bash : langage de script Unix/Linux.
- H2 Database : base de données en mémoire pour les tests.

# Conclusion :

Ce TP consolide la compréhension du fonctionnement interne d’une application Spring Boot et de son intégration dans un environnement système.

L’étudiant acquiert une approche complète du déploiement automatisé, depuis la compilation jusqu’à la supervision.

Cette maîtrise constitue un prérequis essentiel avant d’aborder les concepts de conteneurisation (Docker) et d’orchestration (Kubernetes).
